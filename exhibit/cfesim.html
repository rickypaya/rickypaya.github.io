This project was made during a period before the official training class had begun, where I, as a mentee engineer, could explore and learn about Bloomberg's internal stack hands-on through a project that would have direct imapct on client experience. With help from my new team which I returned to join after completing this project, and subsequently the training class.

Here is a bit more about the project in a technical sense, starting with the context of the current standing codebase. ETMM, one of many "funtions" that live in the Bloomberg terminal, is managed by a team, currently standing at 9 engineers. This application involves one main traiding user interface, along with nine backend repositories, each uniquely designed for a specific purpose, whether it be controlling some aspect of the main application or communicating with one of its dependent services such as the Pricing Engine and Quoting Engine services. The service in focus for this project was the Custom Formula Engine service or CFESVC for short. This service was fairly new for the application system, so it was ripe for innovation. This service's main responsibility is to input, parse, save, subscribe, and calculate formulas and publish those values back out to the main trading screen onto a dedicated column for each security. Then, clients would have the opportunity to use custom excel-like formulas to price their securities to their prefrences. Though, there remained an issue. When traders would have quoted their securities out to the market, there would be no space on the main interface to experiment with formulas without interferring with those securities being priced to the market. This is where I had an opportunity to innovate.
